DATABASE
================================================================================

Location: payment-gateway/payments.db (SQLite)

Tables:
  merchants
    id (TEXT) - Merchant UUID
    name (TEXT) - Company name
    email (TEXT) - Contact email
    api_key (TEXT) - Secret key
    webhook_url (TEXT) - Notification endpoint
    is_active (INTEGER) - 1 = active, 0 = inactive
    created_at (TEXT) - Timestamp

  payments
    id (TEXT) - Payment UUID
    merchant_id (TEXT) - Merchant reference
    amount (REAL) - Fiat amount
    fiat_currency (TEXT) - USD, EUR, GBP, AED
    crypto_currency (TEXT) - ETH, BTC, USDT
    wallet_address (TEXT) - Destination wallet
    customer_email (TEXT) - Customer email
    customer_name (TEXT) - Customer name
    status (TEXT) - pending, completed, failed, refunded
    provider (TEXT) - 'stripe', 'transak', etc
    provider_order_id (TEXT) - Provider's ID
    created_at (TEXT) - Timestamp
    updated_at (TEXT) - Last updated

Queries:
  SELECT * FROM payments WHERE status = 'completed';
  SELECT COUNT(*) FROM payments WHERE created_at > datetime('now', '-1 day');
  SELECT SUM(amount) FROM payments WHERE status = 'completed';

PAYMENT FLOW
================================================================================

1. Customer initiates payment via API
   POST /api/comprehensive-checkout

2. Server creates payment record in database
   Status: pending

3. Server calls Stripe Checkout Session API
   Returns: Secure checkout URL

4. Customer clicks link and completes payment
   On: Stripe-hosted secure page

5. Stripe confirms payment
   Status: checkout.session.completed

6. Stripe sends webhook to server
   POST /webhooks/stripe

7. Server verifies webhook signature
   Confirms: HMAC-SHA256 valid

8. Server updates payment status
   Status: completed

9. Server notifies merchant
   Via: Webhook + Telegram/WhatsApp

10. Server transfers crypto to wallet
    Via: Provider (MetaMask, Transak, etc)
